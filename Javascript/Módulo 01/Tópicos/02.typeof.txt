O operador typeof existe no Javascript como uma ferramenta de inspeção de valores em tempo de execução. Ele foi criado para permitir que o desenvolvedor identifique rapidamente o tipo de um valor armazenado em uma variável ou retornado por uma expressão, algo especificamente útil em uma linguagem dinÇmica como javascript, onde os tipos não são declarados de forma explícita. Ao usar typeof, o runtime retorna uma string descrevendo a categoria do valor observado, permitindo decisões condicionais e verificações simples de tipo sem precisar recorrer a comparações complexas ou introspecção manual.

Na prática, typeof retorna resultados consistentes para a maioria dos tipos primitivos, para números retorna "number", para strings "string", para booleanos "boolean", para símbolos "symbol", para bigint "bigint", e para valores undefined retorna "undefined". No caso de funções, ele retorna "function", e para objetos em geral, incluindo arrays e objetos literais, retorna "object". É importante perceber que, apesar de retornar uma string, typeof não cria um sistema de tipos formal, ele apenas reflete como o runtime classifica os valores internamente, sem fornecer nenhuma garantia de estrutura ou validação estática.

Apesar de sua utilidade, o typeof possui algumas inconsistências históricas que são fruto da evolução da linguagem. O exemplo mais conhecido é que typeof null retorna "object", que é confuso à primeira vista, já que null é, conceitualmente, um primitivo que representa ausência de valor. Esse comportamento é um legado de versões antigas do Javascript, mas ainda está presente por razões de compatibilidade. Outro caso é que arrays, objetos literais e funções também são classificados de forma genérica, e o operador não consegue diferenciar tipos mais específicos dentro do mesmo grupo, o que limita seu uso quando é necessário conhecimento detalhado da estrutura. 

Enquanto typeof apenas inspeciona valores no momento da execução e não garante nada sobre como eles serão usados, Typescript permite que você defina explicitamente o tipo de variáveis, parâmetros de funções e retornos, garantindo previsibilidade e segurança antes mesmo de rodar o código. Por exemplo, você pode declarar que uma variável idade é um number e que uma função só aceita strings. Se você tentar passar outro tipo, o compilador acusa erro. Isso elimina muitas classes de bugs que em javascript só seriam detectadas em runtime.
